launch(self, component=None, flags=0x04000000, data=None, action=None, uri=None, mimetype=None, categories=None, extras=None, waittime=None)

source code 

Starts an App on device by sending an Intent constructed from the supplied arguments. Also you can start an Activity by touching the icon of APP on home screen.

Arguments:

Parameters:
  • component (String) - The component for the Intent. Using this argument will direct the Intent to a specific class within a specific Android package.
  • flags (iterable) - An iterable data structure containing flags that control how the Intent is handled.
  • data (String) - The data URI for the Intent.
  • action (String) - The action for the Intent.
  • uri (String) - The uri for the Intent.
  • mimetype (String) - The MIME type for the Intent.
  • categories (String) - An iterable data structure containing strings that define categories for the Intent.
  • extras (String) - A dictionary of extra data for the Intent.
  • waittime (Int) - The idle time after launching activity. The default value is 3 seconds. See "runner/stability/sysconfig"
Returns: testworker.Action

touch(self, arg, similarity=0.7, waittime=None)

source code 

Perform a touch event on the touch point or on the screen region. The touch point specified by type to the screen location specified by x and y. If the screen region want to touch not found in the current screen snapshot will throw exception.

Parameters:
  • arg (string) - String : a image file path which contained in the current device's scrren.
  • arg (string) - The screen location specified by x and y. Format: (x,y) the screen location specified by x and y. x The horizontal position of the touch in actual device pixels, starting from the left of the screen in its current orientation. y The vertical position of the touch in actual device pixels, starting from the top of the screen in its current orientation.
  • similarity (float) - The minimum similarity a match should have. If omitted, the default is used
  • waittime (int) - The idle time after touch action. Default value is 1 seconds. See "runner/stability/sysconfig"
Returns: testworker.Action

input(self, content, waittime=None)

source code 

Type the text at the current device's screen focused input field The content the characters contained in message to this device, as if they had been typed on the device's keyboard. This is equivalent to calling press() for each keycode in message using the key event type DOWN_AND_UP.

Parameters:
  • content (string) - A string containing the characters to send. as if they had been typed on the device's keyboard.
  • waittime (int) - The idle time after input action. Default value is len(content)*1/3*0.5 seconds. See "runner/stability/sysconfig"
Returns: testworker.Action

drag(self, start, end, time, steps, waittime=None)

source code 

Perform a drag-and-drop operation by dragging at the given click point. Used to simulate a drag gesture (touch, hold, and move) on this device's screen.

Parameters:
  • start (tuple) - The starting point of the drag gesture, in the form of a tuple (x,y) where x and y are integers.
  • end (tuple) - The end point of the drag gesture, in the form of a tuple (x,y) where x and y are integers.
  • time (int) - The duration of the drag gesture in seconds. The default is 1.0 seconds.
  • steps (int) - The number of steps to take when interpolating points. The default is 10.
  • waittime (int) - The idle time after drag action. Default value is 1 seconds. See "runner/stability/sysconfig"
Returns: testworker.Action
